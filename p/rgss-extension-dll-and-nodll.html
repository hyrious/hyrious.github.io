如何编写 RGSS 扩展插件

本文包含了用其他工具（例如 C）为 RGSS 编写扩展功能（即 DLL）的基本操作流程，编译环境 Windows 10，需要阅读者有基本的 C 语言常识。

编写 DLL

DLL 文件其实就是导出函数给其他程序调用，对于 RGSS，我们需要使用 __stdcall 的调用约定，然后通过以下步骤导出：

a.c

int __stdcall add(int a, int b) {
    return a + b;
}

a.def

EXPORTS
    add

编译

cl /LD a.c /link /def:a.def # Visual Studio
# 或者
clang -m32 -shared a.c -o a.dll -Wl,-def:a.def
# 或者
gcc -m32 -shared a.c -o a.dll

RGSS 调用

Win32API.new('a.dll', 'add', 'll', 'l').call(3, 5) #=> 8

使用 Win32API

Win32API 这个类简单好用，不过支持的数据类型很少，下面用两个例子说明一下使用方式：

签名：int MessageBox(HWND hWnd, LPCTSTR lpText, LPCTSTR lpCaption, UINT uType);

其实很简单，我们看到 lp 开头的参数肯定是 p，其他填 l 即可。

MessageBox = Win32API.new('user32', 'MessageBox', 'LppL', 'l')
MessageBox.call(0, 'content', 'title', 16)

这里有个细节是我用了大写的 L，与小写的区别是它是无符号的。
简单地说，l = i = int, L = I = unsigned，不过其实这里都可以用，没什么影响。

签名：BOOL GetCursorPos(LPPOINT lpPoint);

GetCursorPos = Win32API.new('user32', 'GetCursorPos', 'p', 'L')
GetCursorPos.call(doubleL = "\0" * 8)
point = doubleL.unpack('LL') #=> [12, 34]

常用代码

# 移除末尾空格和 "\0"
"asd \0\0\0".unpack('A*') #=> "asd"
# 移除末尾 "\0"
"asd \0\0\0".unpack('Z*') #=> "asd "
# 添加 "\0"；注意，使用 WideChar 时可能要添加两个 "\0"
["asd"].pack('Z*') #=> "asd\0"
["asd\0"].pack('Z*') #=> "asd\0\0"
# ASCII 范围内的 UTF-8/WideChar 互换
'hello'.unpack('U*').pack('S*') #=> "h\0e\0l\0l\0o\0"
"h\0e\0l\0l\0o\0".unpack('S*').pack('U*') #=> 'hello'
# 强行使用二进制编码（防止 UTF-8 编码报错）
"hello".b

我写了一个 gem 来直接在控制台里测试、调用 dll 函数。

注意点

Win32API 只支持 VPpNnLlIi 这些符号作为第三、四的参数，支持的参数长度只有 0 和 4 字节，因此在设计接口的时候一定要注意这一点，比如 double 什么的就不好直接传参啦，float 也要先转成 L ([3.14].pack('F').unpack('L')[0]) 才能传进去。

实现免 DLL 扩展

警告，以下属于黑科技范畴。

CallWindowProc 函数可以直接运行一个回调函数，利用这一点可以往 RGSS 里塞机器码：

gcc -w -m32 -c -O -fno-ident a.c
objcopy -O binary -j .text a.o a

上面两步生成的文件 a 的内容：

8b44 2408 0344 2404 c3
objdump -M intel -S a.o
00000000 <_Add>:
   0:   8b 44 24 08             mov    eax,DWORD PTR [esp+0x8]
   4:   03 44 24 04             add    eax,DWORD PTR [esp+0x4]
   8:   c3                      ret

传给 CallWindowProc：

CallWindowProc = Win32API.new 'user32', 'CallWindowProc', 'pLLLL', 'L'
CallWindowProc.call [
  0x8b, 0104,0044, 8,  # [1] mov 8(%esp), %eax
  0x03, 0104,0044, 4,  # [0] add 4(%esp), %eax
  0xc2,         16,0,  #     ret $16
].pack('C*'), 3, 5, 0, 0 # => 8

注意：此代码在 RGD 上并不会通过，因为 Windows 对内存有权限管理，一段内存需要有执行权限才能被运行，需要用 VirtualProtect 给一下上面代码 pack('C*') 出来的字符串加一个 buff。

其实，支持回调函数就可以……

# 这段代码来自 http://en.cppreference.com/w/c/algorithm/qsort
compare_ints = [
  %w[ 8b44 2404 8b10 8b44 2408 8b00 39c2 7c07
      0f9f c00f b6c0 c3b8 ffff ffff c390 9090 ].join
].pack('H*')

ints = [ -2, 99, 0, -743, 2, -2147483648, 4 ].pack('l*')
size = 7

dll('msvcrt').qsort(ints, size, 4, compare_ints)
ints.unpack('l*')
#=> [-2147483648, -743, -2, 0, 2, 4, 99]

异步？

由于 GIL 的存在，普通的 Ruby 函数（这里我没有使用方法这个词，因为现在是在调用 DLL 的层面说的）无法简单地传给任何有异步回调参数的接口里，要在 RGSS 中使用，需要自己编写 wrapper 来封装成同步的接口，比如我封装的 Rail API 成就接口。
